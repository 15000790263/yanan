# 施工日志入口页设计

## 概述

将施工日志菜单下的两个子菜单（施工日志填报、历史记录）合并为一个入口页，用户点击施工日志后进入入口页，通过两个大按钮分别进入对应功能。

## 页面结构

```
┌─────────────────────────────┐
│  ◀  施工日志                │
├─────────────────────────────┤
│                             │
│  ┌─────────────────────┐    │
│  │    📝               │    │
│  │   施工日志填报      │    │
│  │  填写今日施工情况   │    │
│  └─────────────────────┘    │
│                             │
│  ┌─────────────────────┐    │
│  │    📋               │    │
│  │   施工历史记录      │    │
│  │  查看历史填报记录   │    │
│  └─────────────────────┘    │
│                             │
└─────────────────────────────┘
```

## UI 设计细节

### 卡片样式
- 白色背景 (`#ffffff`)
- 轻微阴影: `0 2px 8px rgba(0,0,0,0.08)`
- 圆角: `12px`
- 间距: 卡片之间 `16px`
- 卡片尺寸: 宽度 `calc(100% - 32px)`，高度约 `100px`
- 左右边距: `16px`

### 图标
- 使用 Vant 图标或项目已有图标
- 图标大小: `32px`
- 图标颜色: `#1989fa`（Vant 蓝色）

### 文字
- 标题: `16px`，`font-weight: bold`，颜色 `#323233`
- 描述: `13px`，颜色 `#969799`

### 交互效果
- 点击时轻微缩放 (`transform: scale(0.98)`)
- 背景色微变，增强点击反馈

## 技术实现

### 路由调整
- 菜单 URL 从 `/log/daily` 改为 `/log/index`
- 新增 `log-index.vue` 入口页面组件

### 组件创建
- 新文件: `kel-app/src/views/log/components/log-entry.vue`

### 跳转逻辑
- 入口页两个按钮分别跳转到:
  - 施工日志填报 → `/log/construction-log`
  - 施工历史记录 → `/log/log-history`

## 验收标准

1. 点击施工日志菜单进入入口页
2. 入口页显示两个可点击的卡片按钮
3. 点击"施工日志填报"跳转到填报页面
4. 点击"施工历史记录"跳转到历史记录页面
5. UI 风格与现有项目保持一致
